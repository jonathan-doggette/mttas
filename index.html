Splunk implementation notes

Recommended build backlog

1. Confirm dashboard owners

Assign a business owner and technical owner for each OPSTAT dashboard before building SPL searches, alert logic, and permissions.

2. Map data sources

Prioritize Splunk Enterprise, OpenTelemetry, AWS CloudWatch, AWS CloudTrail, ServiceNow, PRTG, PagerDuty, Slack, GitHub/GitLab, Tenable, and border/intelligence feeds.

3. Define metric contracts

Normalize field names, service names, event types, timestamps, severities, owners, and SLAs before dashboard buildout.

4. Build demo indexes first

Use synthetic or masked sample data for stakeholder demos, then replace with production indexes as connectors and ingestion pipelines are approved.

5. Keep workforce views privacy-safe

Use aggregated workflow, ticket, availability, and system telemetry. Do not build keystroke, screenshot, webcam, mouse tracking, message-content, or per-user scoring dashboards.

6. Create alert-to-workflow handoffs

For actionable panels, define ServiceNow ticket creation, PagerDuty escalation, Slack notification, and owner assignment rules.
